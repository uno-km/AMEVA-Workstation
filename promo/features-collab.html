CRDT Sync

Y.js CRDT 기반 실시간 다자간 문서 공유

복잡한 중앙 분쟁 조정 알고리즘 없이, 로컬 단말들끼리 랙 없는 병합 상태를 유지합니다.

동시 편집의 비밀: Y.js

기존의 협업 에디터들은 중앙 서버의 판단 하에 텍스트 인덱스를 재조정하는 OT(Operational Transformation) 방식을 사용하여 랙이 심하고 서버 의존도가 높았습니다. AMEVA는 데이터 분산 병합을 보장하는 **CRDT(Conflict-free Replicated Data Type)** 모델을 적용하여:

- 각 사용자의 커서 위치와 하이라이트 영역이 개별 노드 ID로 매핑되어 데이터 오버헤드 없이 스트리밍됩니다.
- 인터넷 전송이 불가능한 보안 격리 연구실 내에서도, 특정 컴퓨터의 포트(1234)를 방화벽 허용하는 것만으로 완벽한 독자적 로컬 협업 서버가 기동됩니다.
